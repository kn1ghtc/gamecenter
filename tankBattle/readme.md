# Tank Battle Game - 面向对象重构版

## 概述

这是一个完全重构的坦克大战游戏，采用面向对象设计，具有更好的扩展性和可维护性。

## 新特性

### 🎯 胜利条件改进
- **双重胜利条件**: 需要同时摧毁敌方基地并消灭所有敌人才能通关
- **配置化**: 可通过 `config.py` 中的 `WIN_CONDITION` 轻松修改胜利条件
- **敌方基地**: 每关都有一个需要摧毁的敌方基地

### 🔫 多种子弹类型
1. **普通弹** (NORMAL)
   - 标准伤害和速度
   - 无特殊效果

2. **穿甲弹** (PIERCING)
   - 可以穿透墙壁
   - 更高的伤害和速度
   - 黄色标识

3. **爆炸弹** (EXPLOSIVE)
   - 爆炸范围伤害
   - 较慢但伤害更高
   - 橙色标识，带闪烁效果

4. **掩体弹** (BARRICADE) ⭐ **新增**
   - 碰撞后在撞击位置生成防御墙
   - 可为玩家提供战术掩护
   - 有冷却时间限制，需策略性使用
   - 绿色标识

### 🤖 增强AI系统 ⭐ **新增**
- **领域感知**: AI坦克根据自身位置智能切换攻防模式
  - **防守模式**: 在己方基地附近时，优先攻击玩家，保持警戒距离
  - **攻击模式**: 突破到玩家领域时，优先攻击玩家基地，快速推进
- **动态目标切换**: 根据威胁等级和距离智能选择攻击目标
- **战术移动**: 不同模式下采用不同的移动策略和射击频率
- **快速决策系统** ⭐ **最新优化**:
  - **智能障碍处理**: 快速判断攻击障碍物还是绕行，避免原地调整角度
  - **角度优化**: 放宽瞄准精度要求，提高射击频率和移动效率
  - **超时机制**: 防止AI陷入无效循环，确保始终保持行动
  - **紧急行动**: 被困时自动执行强制行动，突破僵局
  - **高频作战**: 战斗状态下提高射击频率和反应速度
- **地图生成优化** ⭐ **新增优化**:
  - **基地保护**: 地图生成时自动清除基地位置的围墙，避免基地被埋没
  - **边缘优化**: 屏幕边缘2格范围内不生成围墙，防止边缘卡死问题
  - **智能碰撞**: 坦克在屏幕边缘或近距离遇到障碍物时自动攻击清除
  - **边缘检测**: AI和玩家坦克都具备10像素范围的障碍物检测和攻击能力
- **边缘行为优化** ⭐ **最终优化**:
  - **智能后退**: AI坦克到达屏幕边缘时主动后退调整而不是原地转向
  - **快速脱困**: 边缘被困时立即触发后退机制，避免无效的角度调整
  - **方向识别**: 准确识别边缘方向并选择最佳后退路径
  - **角落处理**: 特别处理屏幕四个角落的后退策略
- **智能目标系统** ⭐ **核心优化**:
  - **模式驱动**: 根据AI模式（防守/攻击）智能选择攻击目标
  - **反击机制**: 被玩家攻击时立即反击，不受模式约束
  - **精准障碍清理**: 只攻击阻挡目标路径的障碍物，避免盲目清墙
  - **目标优先级**: 防守模式优先攻击玩家，攻击模式优先攻击基地

### ⚙️ 配置化系统
- **独立配置文件**: `config.py` 包含所有游戏参数
- **玩家设置**: 可单独配置玩家坦克属性（生命值、速度、子弹类型等）
- **敌方设置**: 可单独配置敌方坦克属性
- **子弹配置**: 每种子弹类型都可独立配置

### 🏗️ 面向对象设计
- **模块化**: 代码拆分为多个专门的模块
- **可扩展**: 新功能易于添加
- **可维护**: 清晰的类结构和职责分离

## 文件结构

```
tankBattle/
├── main_oop.py          # 新的主游戏文件
├── config.py            # 游戏配置文件
├── tank_system.py       # 坦克系统（玩家和敌方坦克）
├── bullet_system.py     # 子弹系统（多种子弹类型）
├── environment.py       # 环境系统（墙壁、基地）
├── level_manager.py     # 关卡管理器
├── main.py             # 原版游戏文件（保留）
└── assets/             # 游戏资源
    ├── level*.map      # 关卡地图文件
    └── ...
```

## 游戏操作

- **↑/↓**: 前进/后退
- **←/→**: 左转/右转
- **空格**: 射击
- **TAB**: 切换子弹类型 ⭐ **新增**
- **P**: 暂停/继续
- **回车**:
  - 关卡完成时：进入下一关
  - 游戏结束时：重新开始

### 子弹类型切换 ⭐ **新增**
- 按 **TAB** 键可在不同子弹类型间循环切换
- 当前子弹类型会在炮口颜色上体现：
  - 白色：普通弹
  - 黄色：穿甲弹
  - 橙色：爆炸弹
  - 绿色：掩体弹
- - 掩体弹有冷却时间，冷却期间无法发射

## 配置说明

### 修改玩家配置
在 `config.py` 中修改 `PLAYER_CONFIG`:
```python

    'HEALTH': 100,        # 生命值
```

## 游戏机制

### 🧱 隔离围墙系统 ⭐ **调整**
- **中央隔离带**: 地图中央自动生成2排特殊隔离围墙，将战场分为两个势力范围
- **可被攻击**: 生命值20；可被常规攻击摧毁；穿甲弹仍可穿透
- **智能绕行**: AI坦克遇到隔离围墙时会自动寻找通道或执行绕行策略
- **随机通道**: 每次生成地图时随机预留3个通道供坦克穿行，增加战术变化
- **穿甲穿透**: 穿甲子弹可以穿过隔离围墙进行攻击，提供远程压制能力
- **特殊外观**: 深蓝色带条纹的特殊外观，明显区别于普通围墙
- **战术意义**:
  - 限制坦克直接冲击，增加战术深度
  - 通道位置成为争夺的战略要点
  - 穿甲弹的价值大幅提升

### 关卡结构
- **玩家基地**: 蓝色基地，位于屏幕底部，需要保护
- **敌方基地**: 紫色基地，位于屏幕顶部，需要摧毁
- **迷宫墙壁**: 可以被射击摧毁的障碍物
- **隔离围墙**: 中央的特殊围墙，不可摧毁，穿甲弹可穿透 ⭐ **新增**
- **敌方坦克**: 会主动攻击玩家和玩家基地

### 胜利条件
默认情况下，需要同时：
1. 摧毁敌方基地
2. 消灭所有敌方坦克

### 失败条件
以下任一情况会导致游戏失败：
1. 玩家坦克被摧毁
2. 玩家基地被摧毁

## 扩展指南

### 添加新的子弹类型
1. 在 `config.py` 的 `BULLET_TYPES` 中添加新类型
2. 在 `bullet_system.py` 的 `Bullet.draw()` 方法中添加绘制逻辑
3. 如需特殊效果，在 `Bullet` 类中添加相应方法

### 添加新的坦克类型
1. 在 `tank_system.py` 中继承 `BaseTank` 类
2. 在 `config.py` 中添加相应配置
3. 在关卡生成时使用新的坦克类型

### 修改AI行为
在 `tank_system.py` 的 `EnemyTank.update_ai()` 方法中修改敌方AI逻辑。
